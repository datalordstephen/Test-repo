Process of using Git

1.) Cd into the project directory and initialize it - git init
2.) Add the repo files you want to push to github - git add ....
3.) Commit the changes -  git commit -m "First commit"
4.) Connect to the github repo - git remote add origin https://github.com/datalordstephen/repo-name.git
5.) Make branch name "main" - git branch -M main (Optional)
6.) Push to the github repo - git push -u origin main

Iterating
After editing the repo files, when pushing again
1.) Check status to verify that the changes were tracked, but are unstaged (havent been re-added)
2.) Stage files - git add ...
3.) commit files - git commit -m "Edited the button"
4.) Push to github repo - git push -u origin main(or master)

Branches
A branch is created in order to modify a file without affecting the original
1.) Creating a new branch - git checkout -b "name"
2.) Staging and commiting the new branch - same way as above
3.) Switching back to the main branch - git checkout main
4.) merging the test branch with the main branch - git merge test
5.) Pushing the test branch or main branch to github - git push -u origin main/test
